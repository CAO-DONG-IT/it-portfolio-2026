# Java 基礎メモ（java-basics）

## 位置づけ
Javaは現状一時停止中だが、ConsoleTodo作成時に整理した基礎知識（入力・分岐・例外・リスト操作）を再利用できる形で残す。

## 関連
- ConsoleTodo：`java/ConsoleTodo/`
- 学習ログ：`weekly-log/`

## Scanner（標準入力）の基本
- `Scanner sc = new Scanner(System.in);`
  - **意味**：キーボード入力（標準入力）を読むための Scanner を用意する。
  - `System.in` は **入力元（キーボード）** を表す。

- `int a = sc.nextInt();`
  - **意味**：入力値を **int（整数）として読み取り**、変数 `a` に代入する。
  - 数字以外が来ると例外が起きる可能性がある（`try-catch` で対策できる）。

- `String day = sc.nextLine();`
  - **意味**：入力された **1行分を文字列として** 読み取る（改行まで）。
  - `nextInt()` の後に `nextLine()` を使うと **改行が残る** ことで意図しない読み取りになる場合がある。

## `new` の意味
- `new` は **インスタンス生成（オブジェクトを作る）**。
- `Scanner` や `ArrayList` はクラス（設計図）なので、そのままでは使えない。
  - `new Scanner(System.in)`：Scanner を実体化
  - `new ArrayList<>()`：リストを実体化

## リテラルと変数
- **リテラル**：プログラム中に直接書く値
  - 例：`10` / `"水"` / `'A'` / `true`
- **変数**：値を入れて名前で扱う箱
  - 例：`int a = 10;` の `a`
- 使い分けの目安：
  - 「その場で一回だけ使う固定値」→ リテラル
  - 「後で再利用／変更したい値」→ 変数

## メソッド：引数あり／引数なしの判断（目安）
### 判断ルール
- **引数なし**：メソッド内部で必要な情報が揃っている（または固定処理だけ）
  - 例：定型表示、状態表示 など
- **引数あり**：呼び出し側からデータを渡さないと処理できない
  - 例：合計を出す、文字列を登録する、指定番号を削除する など

### ConsoleTodo での例
- 引数なし（または最小）
  - `showMenu()`：メニュー表示だけなら引数なしで成立しやすい
- 引数あり（必要なデータを渡す）
  - `showMenuAndRead(Scanner sc)`：入力を読むには `Scanner` が必要
  - `addTask(Scanner sc, ArrayList<String> tasks)`：入力（sc）と保存先（tasks）が必要
  - `listTasks(ArrayList<String> tasks)`：一覧対象（tasks）が必要
  - `delTask(Scanner sc, ArrayList<String> tasks)`：削除番号入力（sc）と対象（tasks）が必要

## switch の要点（Stringでも使える）
- `switch(day)` の `day` が `"月" "火" ...` のどれかで分岐する。
- `break` を忘れると次の `case` へ落ちる（意図しない動作になりやすい）。
- `default` は「どれにも当てはまらない入力」の受け皿。

## 入力チェック（半角/全角・英字・範囲外）
- 仕様として「半角数字 `0/1/2/3` のみ許可」などに統一すると挙動が安定する。
- 例：
  - 全角 `１` → 不正入力として扱う
  - `a` → 不正入力
  - `4` → メニュー範囲外

## try-catch（例外処理）の最低限
- 目的：入力が数字でない等の「想定外」が来てもアプリを落とさず、メッセージを出して続行する。
- イメージ：
  - `try`：失敗する可能性がある処理（例：数字変換／読み取り）
  - `catch`：失敗したときの受け皿（例：`数字で入力してください`）

## 画面表示の整形（タブ）
- 文字列に `\t` を入れると **タブ区切り** で揃えやすい。
- 例：`"1)追加\t2)一覧\t3)削除\t0)終了"`
