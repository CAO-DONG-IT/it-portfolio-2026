# Java 基礎メモ（java-basics）

## Scanner（標準入力）の基本
- `Scanner sc = new Scanner(System.in);`
  - **意味**：キーボード入力（標準入力）を読むための「読み取り担当（Scanner）」を作る。
  - `System.in` は **入力元（キーボード）** を表す。

- `int a = sc.nextInt();`
  - **意味**：入力された値を **int（整数）として読み取って** 変数 `a` に入れる。
  - 数字以外が来ると例外が起きる可能性がある（後述の `try-catch` で対策できる）。

- `String day = sc.nextLine();`
  - **意味**：入力された **1行分を文字列として** 読み取る（改行まで）。
  - `nextInt()` の後に `nextLine()` を使うと「改行が残る」問題が出ることがあるので注意（入力読み取り順を意識する）。

## `new` の意味
- `new` は **インスタンス生成（オブジェクトを作る）**。
- `Scanner` や `ArrayList` は「設計図（クラス）」なので、そのままでは使えない。
  - `new Scanner(System.in)`：Scanner を実体化
  - `new ArrayList<>()`：リストを実体化

## 字面量（リテラル）と変数
- **リテラル（字面量）**：プログラム中に直接書く値
  - 例：`10` / `"水"` / `'A'` / `true`
- **変数**：値を入れて名前で扱う箱
  - 例：`int a = 10;` の `a`
- 使い分け：
  - 「その場で一回だけ使う固定値」→ リテラル
  - 「後で再利用/変更したい値」→ 変数

## メソッド：引数あり／引数なしの判断（肌肉記憶）
### 判断ルール
- **引数なし**：メソッド内部で必要な情報が揃っている（または固定処理だけ）
  - 例：画面に定型表示する、状態を表示する、など
- **引数あり**：呼び出し側から「材料（データ）」を渡さないと処理できない
  - 例：合計を出す、指定された文字列を登録する、指定番号を削除する、など

### ConsoleTodo での対応イメージ
- 引数なし（または最小）
  - `showMenu()`：メニュー文言の表示だけなら引数なしで成立しやすい
- 引数あり（必要な材料を渡す）
  - `showMenuAndRead(Scanner sc)`：入力を読むには `Scanner` が必要
  - `addTask(Scanner sc, ArrayList<String> tasks)`：入力（sc）と保存先（tasks）が必要
  - `listTasks(ArrayList<String> tasks)`：一覧表示する対象（tasks）が必要
  - `delTask(Scanner sc, ArrayList<String> tasks)`：削除番号の入力（sc）と対象（tasks）が必要

## switch の要点（Stringでも使える）
- `switch(day)` の `day` が `"月" "火" ...` のどれかで分岐する。
- `break` を忘れると次の `case` へ落ちる（意図しない動作になる）。
- `default` は「どれにも当てはまらない入力」の受け皿。

## 入力バリデーション（半角/全角・英字・範囲外）
- 仕様として「半角数字 `0/1/2/3` のみ許可」などに統一すると、挙動が安定する。
- 例：
  - 全角 `１` → 不正入力として扱う
  - `a` → 不正入力
  - `4` → メニュー範囲外

## try-catch（例外処理）の最小理解
- 目的：**入力が数字でない** など「想定外」が来てもアプリを落とさずにメッセージを出して続行する。
- イメージ：
  - `try`：失敗する可能性がある処理（例：数字変換/読み取り）
  - `catch`：失敗したときの受け皿（例：`数字で入力してください`）

## 画面表示の整形（タブ）
- 文字列に `\t` を入れると **タブ区切り** で揃えやすい。
- 例：`"1)追加\t2)一覧\t3)削除\t0)終了"`
