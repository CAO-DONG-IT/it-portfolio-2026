# ARP（Address Resolution Protocol）（JST 2026-01-07）

## 用語（JP / EN / 中文）
- ARP / Address Resolution Protocol / 地址解析协议
- IPアドレス / IP address / IP地址
- MACアドレス / MAC address / 物理地址
- ブロードキャスト / broadcast / 广播
- ブロードキャストドメイン / broadcast domain / 广播域
- デフォルトゲートウェイ / default gateway / 默认网关
- ARPテーブル（キャッシュ）/ ARP table (cache) / ARP表（缓存）

## 何をする仕組みか
- **ARP は「IPアドレス → MACアドレス」を解決する仕組み**。
- 同一LAN内（同一ブロードキャストドメイン）でフレームを届けるには、宛先MACアドレスが必要になる。

## いつ使うか（代表例）
- 同一LAN内の相手に送るとき：相手のMACを ARP で解決する
- インターネット宛通信のとき：最初に **デフォルトゲートウェイのMACアドレス** を ARP で解決する  
  （フレームの宛先MACは「相手サーバ」ではなく「ゲートウェイ」になる）

## 流れ（最小）
1. 送信側が ARPテーブル を確認（すでにMACが分かればそれを使う）
2. 無ければ ARP Request（ブロードキャスト）を送る
3. 対象ホストが ARP Reply（ユニキャスト）でMACを返す
4. 送信側は ARPテーブル に学習し、以降はキャッシュを利用する

## 注意点
- ARP が届く範囲は基本的に **同一ネットワーク（同一ブロードキャストドメイン）**。
- ルータはブロードキャストを転送しないため、ARP は通常ルータを越えない。

## ひとこと（面接での説明用）
- 「同一LANで通信するには、IPだけでなくMACが必要なので、ARPでIP→MACを解決します」
