# Subnetting Practice（CIDR）

目的：サブネット計算を「毎回同じ手順」で解けるようにし、面接でも説明できる形で残す。  
（結論：**CIDR → ブロックサイズ → 範囲 → NW/BC/有効ホスト** を固定手順にする）

## 用語（JP / EN / 中文）
- プレフィックス長 / prefix length / 前缀长度（例：/28）
- サブネットマスク / subnet mask / 子网掩码
- ネットワークアドレス / network address / 网络地址（ホスト部が全0）
- ブロードキャストアドレス / broadcast address / 广播地址（ホスト部が全1）
- 有効ホスト範囲 / usable host range / 可用主机范围（通常 NW+1 ～ BC-1）
- ブロックサイズ / block size / 块大小（256-マスク値）
- VLSM / Variable Length Subnet Mask / 可变长子网掩码

---

## 解法テンプレ（固定手順）

### A) まず確認する（CIDR とマスク）
1. CIDR を確認（例：`/28`）
2. サブネットマスクに変換（例：`/28 = 255.255.255.240`）
   - `/24` までは 255.255.255.0
   - それ以降は「最後の1オクテット」を計算する

### B) ブロックサイズで範囲を出す（最後の1オクテット）
- `255.255.255.X` のとき：**ブロックサイズ = 256 - X**
- 範囲（サブネットの区間）は「ブロックサイズ刻み」
- その範囲に対象IPが入る区間が、該当サブネット

### C) NW / BC / 有効ホスト
- NW（ネットワークアドレス）= 範囲の先頭
- BC（ブロードキャスト）= `NW + ブロックサイズ - 1`
- 有効ホスト範囲 = `NW + 1 ～ BC - 1`
- ホスト数 = `2^(ホスト部ビット数) - 2`

| --- | --- | --- |

| /26 | 192 | 64 |

| /29 | 248 | 8 |
| /30 | 252 | 4 |

## 例題（書き方の見本）

### Q0（見本）：192.168.10.66/28
- 1) CIDR：`/28` → マスク：`255.255.255.240`
- 2) ブロックサイズ：`256 - 240 = 16`
- 3) 範囲：`0-15 / 16-31 / 32-47 / 48-63 / 64-79 / 80-95 ...`
  - `66` は `64-79` に入る
- 4) NW：`192.168.10.64`
- 5) BC：`192.168.10.79`
- 6) 有効ホスト：`192.168.10.65 ～ 192.168.10.78`
- 7) ホスト数：`2^4 - 2 = 14`
